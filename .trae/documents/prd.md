## 1. 产品概述

Live2D 模型管理面板是一个集成于现有 Live2D API 仓库的前端管理模块，为管理员提供直观的模型查看、管理和上传功能。目标用户为 Live2D API 的运维人员，核心价值在于将原本需要手动编辑 JSON 和文件系统操作的工作流程，转化为可视化的 Web 界面操作。

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 管理员 | 直接访问 | 模型查看、增删改查、文件上传 |

### 2.2 功能模块

1. **模型列表页**：模型卡片网格展示、分组筛选、搜索、Live2D 实时预览
2. **模型详情页**：模型配置查看、文件列表、纹理预览、动作列表
3. **模型管理**：新增模型、编辑模型信息、删除模型
4. **模型上传**：ZIP 包上传、单文件上传、自动生成配置

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 模型列表 | 侧边栏导航 | 分组筛选、搜索框、统计概览 |
| 模型列表 | 模型卡片网格 | 展示模型名称、分组、纹理数、状态标签，点击预览 |
| 模型列表 | Live2D 预览面板 | 侧滑面板，实时渲染 Live2D 模型 |
| 模型详情 | 基本信息卡片 | 模型名称、分组、描述、配置信息 |
| 模型详情 | 文件浏览器 | 模型文件树形展示、文件大小、纹理缩略图 |
| 模型管理 | 新增模型对话框 | 输入 Group/Model 名称和描述，选择已有目录 |
| 模型管理 | 编辑模型对话框 | 修改模型名称和描述信息 |
| 模型管理 | 删除确认对话框 | 确认是否同时删除文件 |
| 模型上传 | 上传对话框 | 拖拽上传 ZIP 或单文件，显示上传进度 |

## 3. 核心流程

用户打开管理面板 → 查看模型列表 → 点击模型卡片查看详情/预览 → 执行管理操作（新增/编辑/删除）→ 上传新模型文件 → 系统自动更新 model_list.json

## 4. 用户界面设计

### 4.1 设计风格

- 主色调：深色系（#1a1b2e 暗蓝黑）+ 赛博朋克风格霓虹青色（#00f0ff）作为强调色
- 次要强调色：柔粉紫（#c77dff）
- 按钮风格：圆角微光按钮，hover 时发光效果
- 字体：Noto Sans SC（中文）+ JetBrains Mono（代码/数据）
- 布局风格：左侧固定导航栏 + 右侧内容区，卡片式布局
- 图标风格：线性图标，霓虹风格

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI 元素 |
|----------|----------|---------|
| 模型列表 | 侧边栏 | 深色背景、分组列表、搜索框、统计数字 |
| 模型列表 | 卡片网格 | 半透明卡片、模型缩略图、状态标签、hover 发光 |
| 模型列表 | 预览面板 | 右侧滑出面板、Live2D Canvas、控制按钮 |
| 模型详情 | 信息卡片 | 深色卡片、配置 JSON 高亮、文件树 |
| 模型管理 | 对话框 | 毛玻璃背景、表单输入、确认按钮 |
| 模型上传 | 上传区域 | 虚线边框拖拽区、进度条、文件列表 |

### 4.3 响应式设计

- 桌面优先设计（≥1200px）：侧边栏 + 内容区双栏布局
- 平板适配（768-1199px）：侧边栏折叠为图标模式
- 移动端适配（<768px）：侧边栏隐藏，汉堡菜单触发，卡片单列
